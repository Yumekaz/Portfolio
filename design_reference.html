Engineering Notes

What I Got Wrong About fsync()

Mini Redis project | 5th semester

Built a write-ahead log for persistence. Assumed fsync() would be fast enough. It wasn't.

What failed: Crash recovery took ~500ms on a 10MB log. Expected buffered writes to help. Linux file system durability semantics made it fragile.
Insight: Persistence has real costs. File I/O is not free. Durability boundaries matter.

Signal Propagation Across Process Boundaries

Mini container runtime | 5th semester

Tried to build process isolation using cgroups and namespaces. Thought containers were straightforward. They're not.

What failed: Signals sent to PID 1 didn't cascade to child processes. SIGTERM didn't propagate. Container shutdown was fragile.
Constraint: Signal handling across namespace boundaries requires careful init process design.
